## Streamlit + FastAPI + Docker + AWS EC2 기반 추천 웹 애플리케이션 구현

---

## 1. 과제 목적

이번 과제는 지금까지 학습한 내용을 종합하여,

**Streamlit 프론트엔드**, **FastAPI 백엔드**, **Docker 컨테이너**, **AWS EC2 배포**를 하나의 프로젝트로 연결해 보는 통합 실습입니다.

학생들은 이번 과제를 통해 다음 내용을 경험하게 됩니다.

- Streamlit을 이용한 사용자 입력 인터페이스 구성
- FastAPI를 이용한 추천 결과 처리 API 구현
- Streamlit과 FastAPI 간 HTTP 통신
- Docker를 이용한 애플리케이션 실행 환경 구성
- AWS EC2 환경에서의 서비스 실행
- 프론트엔드와 백엔드가 연결된 하나의 웹 서비스 구현

---

## 2. 과제 개요

이번 프로젝트에서는 **추천(recommendation)** 형태의 웹 애플리케이션을 구현합니다.

즉, 사용자가 Streamlit 화면에서 몇 가지 정보를 입력하면,

FastAPI가 해당 입력을 받아 처리한 뒤,

추천 결과를 반환하고,

그 결과를 다시 Streamlit 화면에서 보여주는 구조여야 합니다.

### 전체 구조

- **Streamlit**: 사용자 입력 받기 / 결과 출력
- **FastAPI**: 입력값을 받아 추천 결과 생성 후 반환
- **Docker**: 실행 환경 구성
- **AWS EC2**: 실제 실행 환경

즉, 반드시 아래 흐름이 구현되어야 합니다.

**사용자 입력 → Streamlit → FastAPI 호출 → 추천 결과 반환 → Streamlit 화면 표시**

---

## 3. 주제

이번 프로젝트는 **추천 앱**으로 구현해야 합니다.

단, **추천의 세부 주제는 자유**입니다.

즉, 모든 학생이 “추천” 구조는 공통으로 가져가되,

무엇을 추천할지는 각자 자유롭게 정할 수 있습니다.

### 가능한 예시 주제

아래는 예시이며, 반드시 이 중에서 고를 필요는 없습니다.

- **전공 과목 추천 앱**: 관심 분야, 학년, 선호 난이도를 입력하면 추천 과목을 반환
- **학습 방법 추천 앱**: 공부 시간, 선호 학습 방식, 목표를 입력하면 적합한 학습 전략을 추천
- **운동 루틴 추천 앱**: 운동 목적, 주당 가능 횟수, 운동 장소를 입력하면 루틴을 추천
- **여행지 추천 앱**: 예산, 여행 기간, 선호 활동을 입력하면 여행 스타일 또는 여행지를 추천
- **책 추천 앱**: 선호 장르, 관심 키워드, 독서 시간을 입력하면 책 유형을 추천

## 4. 필수 구현 조건

### (1) Streamlit 프론트엔드

Streamlit은 반드시 다음 역할을 해야 합니다.

- 사용자 입력 받기
- 추천 요청 버튼
- FastAPI에 요청 보내기
- FastAPI의 응답 결과를 받아 화면에 표시하기

즉, Streamlit은 단순한 정적 화면이 아니라 **입력과 출력이 있는 프론트엔드**여야 합니다.

---

### (2) FastAPI 백엔드

FastAPI는 반드시 다음 역할을 해야 합니다.

- Streamlit이 보낸 입력값 받기
- 입력값을 바탕으로 추천 결과 생성하기
- 추천 결과를 JSON 형태로 반환하기

추천 로직은 복잡할 필요는 없습니다. 이 로직이 얼마나 완성도 있는지는 채점 기준에 들어가지 않습니다.

if - else 위주의 규칙 기반(rule-based) 추천이어도, 심지어 random 추천도 충분합니다.

예를 들어:

- 입력 조건에 따라 다른 추천 문구를 반환
- 점수 합산 후 유형에 따라 추천 반환
- 카테고리 조합에 따라 추천 리스트 반환

등의 방식이 가능합니다.

### (3) Streamlit → FastAPI 호출 구조 필수

이번 과제의 핵심은 **Streamlit과 FastAPI가 실제로 연결되어 있어야 한다는 점**입니다.

즉,

- Streamlit 안에서 추천 결과를 직접 계산하면 안 되고,
- 반드시 FastAPI에 요청을 보내고,
- FastAPI가 처리한 결과를 받아와서 표시해야 합니다.

---

### (4) Docker 환경 구성

프로젝트는 Docker 환경에서 실행 가능해야 합니다.

- Docker 사용은 필수입니다.
- Streamlit과 FastAPI를 각각 분리하여 다른 컨테이너로 실행하는 것을 권장합니다.

---

### (5) AWS EC2 실행

프로젝트는 반드시 **수업에서 제공된 AWS Learner Lab의 EC2 환경**에서 실행되어야 합니다.

외부에서 EC2 주소로 접속했을 때,

- Streamlit 화면이 열리고
- 추천 기능이 동작해야 합니다.

즉, 단순히 로컬에서만 동작하면 충분하지 않고,

반드시 **EC2 환경에서 실행된 결과**가 있어야 합니다.

## 5. 주제 및 구성 관련 유의사항

이번 과제에서는 **추천 주제와 입력 문항, 결과 방식이 개인별로 특색 있게 구성**되어야 합니다.

즉,

- 추천 대상
- 입력 항목
- 추천 기준
- 결과 표현 방식

등에서 개인별 차별성이 드러나야 합니다.

단순히 주제 이름만 바꾸고

전체 UI 구조, 입력 형식, 코드 구조, 추천 로직이 다른 학생과 과도하게 유사한 경우에는 표절로 판단될 수 있습니다.
